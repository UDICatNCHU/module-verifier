# /add-module - 新增領域模組

從教務處網頁 URL 擷取模組課程資料，新增到 `modules_data.json`，並驗證正確性。

## 使用方式

```
/add-module <URL>
```

範例：
```
/add-module https://oaa.nchu.edu.tw/zh-tw/unit-page-p.342/page-detail.3269
```

## 執行步驟

### Step 1: 擷取網頁資料

使用 WebFetch 擷取 URL，提取以下資訊：

```
prompt: 請完整擷取這個模組的所有資訊，包括：
1. 模組名稱（中英文）
2. 主責教學單位、隸屬學院
3. 召集人
4. 預計施行時間
5. 課程規劃內容：每門課的課程名稱（中英文）、學分數、開課單位、備註（基礎/核心/應用課程、選修規則等）
6. 認證要求：取得認證需修習總課程數、總學分數
7. 修課指引或特殊規則說明
請保留所有原始文字，不要省略任何課程或備註。
```

### Step 2: 確認擷取結果

向使用者展示擷取到的模組結構摘要，請使用者確認：
- 模組名稱
- 課程列表（名稱、學分、層級）
- 認證要求
- 特殊規則

等使用者確認後再進行下一步。

### Step 3: 判斷備註格式

根據課程的層級和選修規則，決定每門課的 `備註` 欄位值。遵循既有的備註 pattern（由 `src/remark-parser.ts` 解析）：

| 情境 | 備註格式 | 範例 |
|------|---------|------|
| 該層級只有 1 門課 | `基礎課程` / `核心課程` | 只有一門基礎課 → `基礎課程` |
| 該層級全部必修 | `基礎課程` / `核心課程` | 3 門核心全部要修 → 每門都標 `核心課程` |
| N 選 M | `核心課程(N選M)` | 三選一 → `核心課程(三選一)` |
| N 選 M + 學分限制 | `核心課程 (N門選M門、修習至少X學分)` | `核心課程 (三門選兩門、修習至少6學分)` |
| 附加限制說明 | 加在括號內 | `基礎課程 (只承認管理學院所開設課程)` |
| 選修兩學期 | 加 `選修兩學期` | `應用課程(三選一) 選修兩學期` |
| 無備註（全部必修） | 留空不填 | 歷史系全部必修的情況 |

中文數字對照：1=一, 2=二, 3=三, 4=四, 5=五, 6=六, 7=七, 8=八, 9=九, 10=十

### Step 4: 建構模組 key

模組 key 格式：`{主責教學單位}_{模組中文名稱}`

範例：`企業管理學系_商業智慧`、`生命科學系_微生物科技`

### Step 5: 檢查是否已存在

讀取 `modules_data.json`，檢查該 key 是否已存在。如已存在，詢問使用者是否要覆蓋。

### Step 6: 寫入 modules_data.json

使用 Node.js 腳本將模組加入 JSON：

```typescript
import { readFileSync, writeFileSync } from 'node:fs'

const data = JSON.parse(readFileSync('modules_data.json', 'utf-8'))

data['<MODULE_KEY>'] = {
  '基本資訊': {
    '中文': '<模組中文名>',
    '英文': '<模組英文名>',
    '領域模組預計開始施行時間': '<施行時間>',
    '主責教學單位': '<主責單位>',
    '主責單位 隸屬一級單位': '<隸屬學院>',
    '合作教學單位': '無 (非必填)',
    '合作單位 隸屬一級單位': '無 (非必填)',
    '召集人': { '<召集人姓名>': '<召集人單位>' }
  },
  '架構計畫': {
    '領域模組名稱': '<模組中文名>',
    '領域模組召集人': '<召集人姓名>',
    '修課指引': '<修課指引原文>'
  },
  '模組總表': {
    '認證要求': {
      '取得認證需修習總課程數': <數字>,
      '取得認證需修習總學分數': <數字>
    },
    '課程規劃內容': [
      // 每門課一筆
      {
        '課程名稱_中文': '<中文名>',
        '課程名稱_英文': '<英文名>',
        '規劃要點': {
          '1': 'U', '2': 'A', '3': 'R', '4': 'S',
          '5': '<學分數字串>',
          '6': '<層級: 1=基礎, 2=核心, 3=應用>'
        },
        '開課單位': '<開課單位>',
        '備註': '<依 Step 3 規則產生的備註>'
      }
    ]
  }
}

writeFileSync('modules_data.json', JSON.stringify(data, null, 2), 'utf-8')
```

### Step 7: 驗證解析正確性

用 Node.js 載入模組並印出解析結果：

```typescript
import { loadModules } from './src/module-loader.ts'

const modules = loadModules('modules_data.json')
const mod = modules.find(m => m.key === '<MODULE_KEY>')

// 印出：模組名稱、認證要求、每個 group 的 label + rule + 課程列表
```

確認：
- 課程數量正確
- group 分組正確（基礎/核心/應用）
- rule type 正確（required / choose_m_from_n / min_credits）
- 認證門檻正確

### Step 8: 更新測試中的模組數量

更新 `test/verifier.test.ts` 中 `loads N modules` 的數字。

### Step 9: 跑測試

```bash
npm test
```

確認所有測試通過。

### Step 10: 報告結果

向使用者報告：
- 新增的模組名稱和 key
- 課程數量和結構摘要
- 認證要求
- 測試結果
- 提醒：如需補課程代碼（科目內碼），可用 `scripts/import-course-codes.ts`

## 注意事項

- 認證要求的數字欄位：如果網頁是文字（如「4門課」），存入 JSON 時用數字 `4`，不要用字串
- 備註格式必須符合 `remark-parser.ts` 能解析的 pattern，否則會被當作 required
- 如果模組有特殊規則（如跨群組依賴、替代課程），需要額外討論處理方式
- 超連結欄位可選填：`'超連結': '<URL>'`
